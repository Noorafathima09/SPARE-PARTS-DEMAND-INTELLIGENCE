
# Spare Parts Demand Intelligence

This project analyzes spare parts sales data from a real-world agency to provide operational demand intelligence. The goal is to support inventory planning, workload management, and branch performance evaluation using Python

---

## Project Overview

- **Business Context:** Spare-parts demand is irregular and service-cycle dependent.
- **Objective:** Transform ERP billing data into actionable insights:
  - Identify fast, medium, and slow-moving items
  - Measure demand stability and volatility
  - Track monthly trends for operational planning
  - Support branch and customer-level decisions

## Tools Used
- Python (data cleaning, analysis, visualization)

## Analysis Highlights
1. Cleaned ERP data & removed invoice total rows
2. Created semantic item categories
3. Classified items as Fast, Medium, Slow
4. Calculated demand stability & volatility
5. Visualized trends & critical items

## Business Impact
- Focus on operationally critical items
- Reduce capital blocked in low-demand inventory
- Manage volatile high-demand items
- Support branch-level planning
